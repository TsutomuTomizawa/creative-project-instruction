# LPコピーパターン

## 概要
セクション別の効果的なコピーパターンと文章構成理論。読みやすさと説得力を両立する体系的アプローチ。

## セクション別コピーパターン

### ファーストビューコピー
```yaml
インパクト型:
  構成: 
    - メイン: 15-25文字
    - サブ: 30-50文字
  トーン: 断定的・力強い
  要素:
    - 数字/実績
    - 変化/結果
    - ベネフィット
  例文構造: 
    - 「たった〇〇で△△を実現」
    - 「〇〇社が選んだ△△」

質問型:
  構成:
    - メイン: 20-30文字（疑問形）
    - サブ: 答えを示唆
  トーン: 共感的・親近感
  要素:
    - 悩み/課題
    - 願望/理想
    - 気づき
  例文構造:
    - 「まだ〇〇で悩んでいませんか？」
    - 「〇〇を変えたいと思いませんか？」

宣言型:
  構成:
    - メイン: 20-30文字
    - サブ: 根拠/証明
  トーン: 自信・確信
  要素:
    - 新提案
    - 革新性
    - 独自性
  例文構造:
    - 「〇〇の常識を変える」
    - 「新しい〇〇の形」
```

### 問題提起/共感コピー
```yaml
共感ストーリー型:
  文字数: 200-300文字
  構成:
    - 状況描写（50文字）
    - 感情表現（100文字）
    - 問題の核心（50文字）
  トーン: 寄り添い・理解
  テクニック:
    - 具体的シーン
    - 感情ワード
    - 「〜ではないでしょうか」

データ提示型:
  文字数: 150-250文字
  構成:
    - 統計提示（50文字）
    - 解釈（100文字）
    - 問題提起（50文字）
  トーン: 客観的・論理的
  テクニック:
    - 信頼できる出典
    - グラフとの連動
    - 比較表現
```

### 解決策提示コピー
```yaml
特徴説明型:
  文字数: 300-400文字
  構成:
    - 概要（100文字）
    - 3つの特徴（各80文字）
    - まとめ（60文字）
  トーン: 明確・簡潔
  テクニック:
    - 箇条書き活用
    - 専門用語の翻訳
    - ビフォーアフター

プロセス説明型:
  文字数: 250-350文字
  構成:
    - 導入（50文字）
    - 3ステップ（各80文字）
    - 結果（60文字）
  トーン: 分かりやすい・順序的
  テクニック:
    - 番号付け
    - 時間軸表現
    - 「たった3ステップ」
```

### 信頼性構築コピー
```yaml
お客様の声型:
  文字数: 150-200文字/件
  構成:
    - 属性（20文字）
    - 課題（50文字）
    - 結果（80文字）
    - 感想（50文字）
  トーン: リアル・具体的
  テクニック:
    - 実名/イニシャル
    - 写真併用
    - 数値入り

実績データ型:
  文字数: 100-150文字
  構成:
    - 見出し（30文字）
    - 数値（50文字）
    - 説明（70文字）
  トーン: 事実ベース・誇張なし
  テクニック:
    - 第三者評価
    - 期間明記
    - 比較優位
```

### オファーコピー
```yaml
価値強調型:
  文字数: 200-300文字
  構成:
    - 通常価格（50文字）
    - 特別価格（50文字）
    - 理由（100文字）
    - 限定性（100文字）
  トーン: お得感・特別感
  テクニック:
    - 削除線価格
    - パーセント表示
    - 「今だけ」「限定」

保証訴求型:
  文字数: 150-200文字
  構成:
    - 保証内容（80文字）
    - 条件（70文字）
    - 安心要素（50文字）
  トーン: 安心・信頼
  テクニック:
    - 「満足保証」
    - 「リスクゼロ」
    - FAQ連動
```

## 文字数理論

### 可読性の原則
```yaml
1行あたり:
  PC: 35-40文字
  SP: 15-20文字

1段落あたり:
  最大: 3-4行
  理想: 2-3行

見出し階層:
  大見出し: 20-30文字
  中見出し: 30-40文字
  小見出し: 40-50文字
```

### セクション別文字数配分
```yaml
全体構成（2000-3000文字）:
  FV: 50-100文字（3%）
  問題提起: 300-400文字（15%）
  解決策: 600-800文字（30%）
  信頼性: 500-600文字（25%）
  オファー: 400-500文字（20%）
  CTA: 100-200文字（7%）
```

## トーン選択マトリクス

### ターゲット別トーン
```yaml
BtoB:
  基調: 専門的・論理的
  避ける: 感情的・カジュアル
  使用語彙: 業界用語OK

BtoC（高額）:
  基調: 丁寧・信頼感
  避ける: 軽薄・押し売り
  使用語彙: 平易な専門用語

BtoC（低額）:
  基調: 親しみ・気軽さ
  避ける: 堅苦しい・難解
  使用語彙: 日常語中心
```

```yaml

  - エビデンス重視

  - 控えめな表現

ライフスタイル:
  - 感情訴求
  - ストーリー性
  - 共感表現

緊急性商材:
  - 行動促進
  - 限定性強調
  - 明確な指示
```
