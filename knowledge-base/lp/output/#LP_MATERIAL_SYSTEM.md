# LP素材管理システム

## 概要
LP制作における素材の体系的な管理手法。役割分類、効果測定、AI活用を統合したシステム。

## 素材役割分類体系

### 機能別分類
```yaml
注目獲得素材:
  役割: 視線を引きつける
  種類:
    - ヒーローイメージ
    - アイキャッチ
    - 動的要素
  効果指標: 
    - 初回視認率
    - 注視時間

情報伝達素材:
  役割: 内容を理解させる
  種類:
    - インフォグラフィック
    - 図解/チャート
    - アイコンセット
  効果指標:
    - 理解度
    - 記憶定着率

感情訴求素材:
  役割: 共感を生む
  種類:
    - 人物写真
    - ストーリー画像
    - 感情表現
  効果指標:
    - 共感度
    - 好感度

信頼構築素材:
  役割: 安心感を与える
  種類:
    - 認証マーク
    - 実績データ
    - 顧客写真
  効果指標:
    - 信頼度向上
    - 不安解消率

行動促進素材:
  役割: アクションを促す
  種類:
    - CTAボタン
    - 矢印/誘導
    - 緊急性表現
  効果指標:
    - クリック率
    - コンバージョン率
```

### セクション別必須素材
```yaml
ファーストビュー:
  必須:
    - メインビジュアル
    - ロゴ/ブランド
    - CTAボタン
  任意:
    - 背景パターン
    - 装飾要素

問題提起セクション:
  必須:
    - 課題ビジュアル
    - 共感イメージ
  任意:
    - グラフ/データ
    - イラスト

解決策セクション:
  必須:
    - 商品/サービス画像
    - 機能説明図
  任意:
    - プロセス図
    - 比較表

信頼性セクション:
  必須:
    - 実績データ
    - 顧客の声素材
  任意:
    - 認証バッジ
    - メディア掲載

オファーセクション:
  必須:
    - 価格表
    - 特典ビジュアル
  任意:
    - 限定表現
    - カウントダウン
```

## 効果×コストマトリクス

### 投資優先度の判定
```yaml
高効果×低コスト（最優先）:
  - ストックフォト活用
  - テンプレート応用
  - 簡易カスタマイズ
  ROI: 200%以上

高効果×高コスト（選択的）:
  - オリジナル撮影
  - カスタムイラスト
  - 動画制作
  ROI: 100-200%

低効果×低コスト（補助的）:
  - 装飾パーツ
  - 背景パターン
  - 汎用アイコン
  ROI: 50-100%

低効果×高コスト（回避）:
  - 過度な演出
  - 複雑なアニメーション
  - 特殊効果
  ROI: 50%以下
```

## AI活用パターン集

### 画像生成AI活用
```yaml
背景/環境画像:
  用途: 雰囲気作り
  プロンプト構造:
    - スタイル指定
    - 色調指定
    - 要素配置
  生成パラメータ:
    - サイズ: 1920x1080
    - スタイル: photorealistic/illustration
    - 品質: high

商品イメージ:
  用途: 使用シーン
  プロンプト構造:
    - 商品特徴
    - 使用環境
    - ターゲット属性
  注意点:
    - ブランドガイドライン遵守
    - 誤解を招かない表現

アイコン/図解:
  用途: 機能説明
  プロンプト構造:
    - flat design
    - minimalist
    - specific color palette
  統一性:
    - 一貫したスタイル
    - 共通カラーパレット
```

### テキスト生成AI活用
```yaml
キャッチコピー候補:
  入力情報:

    - desired tone
  生成数: 10-20案
  選定基準:
    - 文字数適合
    - トーン一致
    - 独自性

顧客の声テンプレート:
  ベース情報:
    - 実際の感想
    - 改善ポイント
    - 属性情報
  加工指針:
    - 自然な表現
    - 具体性追加
    - 感情表現
```

## バリエーション管理手法

### A/Bテスト用バリエーション
```yaml
メインビジュアル:
  バリエーション数: 3-5
  変更要素:
    - 被写体
    - 色調
    - レイアウト
  命名規則: mv_[要素]_[番号]

キャッチコピー:
  バリエーション数: 3-4
  変更要素:
    - 表現アプローチ
    - 文字数
    - トーン
  命名規則: copy_[approach]_[番号]

CTAボタン:
  バリエーション数: 2-3
  変更要素:
    - 文言
    - 色
    - サイズ
  命名規則: cta_[text]_[color]
```

### 素材命名規則
```yaml
基本構造:
  [セクション番号]_[素材タイプ]_[内容]_[バリエーション].[拡張子]

例:
  01_hero_product_a.jpg
  02_icon_problem_01.svg
  03_graph_results_v2.png
  04_photo_customer_b.jpg
  05_table_pricing_test.png

フォルダ構成:
  project_name/
  ├── 00_common/      # 共通素材
  ├── 01_fv/          # ファーストビュー
  ├── 02_problem/     # 問題提起
  ├── 03_solution/    # 解決策
  ├── 04_trust/       # 信頼性
  ├── 05_offer/       # オファー
  ├── 06_cta/         # CTA
  └── 99_archive/     # 不採用素材
```
